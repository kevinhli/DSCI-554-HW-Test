# DSCI 554 Group Project: Pre-Submission

## Description

In your final project, your team will produce a **demo paper** as if submitted to a *visualization* conference. For this, you will write a **research paper**, build and deploy a **demo**, create a **video**, and deliver a **presentation** to the class. The goal of the project is to demonstrate a visual display or system that addresses a visualization problem or challenge. As such, this project demonstrates how to apply class concepts to solve visualization challenges and communicate your contributions professionally.

### Requirements

#### Paper

The paper should be written in **Overleaf** using the [LNCS format](http://www.springer.com/computer/lncs?SGWID=0-164-6-793341-0) format. The paper should be clearly written, concise, and **no more than 5 pages long**. See [PAPER_WRITING_GUIDE.md](PAPER_WRITING_GUIDE.md) for the sections to include.

For this pre-submission, the paper must be completed as follows:

- **Title:** provide a preliminary title
- **Abstract:** Must be fully written.
- **Introduction:** Must be fully written.
- **Related Work:** Must be fully written.
- **Data**: Bullet points.
- **Design and Implementation:** Bullet points.
- **Usage Scenarios and Results:** Bullet points.
- **Conclusion**: Bullet points.
- **References:** Must include major references.

#### Demonstration (Technical Skeleton)

This includes a **rough wireframe in Figma** and the initial deployment of the codebase to GitHub Pages. You must use **Vue.js or React with Bootstrap** and include: 
- **Grid Layout:** Use Bootstrap's grid system to define the dashboard structure.
- **Placeholders:** No data integration is required at this stage. Simply include text placeholders for where the primary visualizations will reside.
- **Deployment:** The deployed demonstration should be publicly accessible on GitHub.io.

#### Project Management

Update **SUBMISSION.md** with the requested information. Team members need to contribute to all artifacts (Paper, Code, Video, and Presentation).
- **Team Lead:** Identify a single team lead for administrative oversight for each artifact.
- **Coordination Plan:** Detail a plan to consolidate weekly reports from team leads on major artifacts to ensure balanced participation and track progress.

### Submission

Update **SUBMISSION.md** with the requested information. Students are expected to:

- Commit the skeleton code base of the demonstration in this repository.
- Complete the [SUBMISSION.md](SUBMISSION.md) file, including the link to GitHub.io, the Overleaf project, and PDF of the paper.
